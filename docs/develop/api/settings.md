# API Настроек

Этот раздел описывает API для управления общими настройками системы.

---

## Обновление настроек

Обновляет одну или несколько общих настроек системы.

**Эндпоинт:** `PUT /rest/settings`

**Метод:** `PUT`

**Аутентификация:** Требуется

### Пример запроса (curl)

```bash
curl -X PUT http://DEVICE_IP/rest/settings \
     -H "Authorization: Bearer YOUR_AUTH_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{
           "setting_key_1": "новое значение",
           "another_setting": 123
         }'
```

### Тело запроса

JSON-объект, содержащий пары "ключ-значение" для настроек, которые необходимо обновить. Система применяет только те настройки, которые присутствуют в запросе.

**Пример тела запроса:**

```json
{
  "setting_key_1": "новое значение",
  "another_setting": 123
}
```

### Ответы

*   **`200 OK`**: Настройки успешно обновлены.
*   **`400 Bad Request`**: Ошибка в формате запроса (например, некорректный JSON) или неверные значения для настроек.
*   **`401 Unauthorized`**: Ошибка аутентификации.
*   **`500 Internal Server Error`**: Внутренняя ошибка сервера при попытке сохранить настройки.
